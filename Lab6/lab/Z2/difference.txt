** Direct **
Producent wysyła wiadomość z określonym kluczem routingu,
a konsumenci oczekujący na wiadomości na konkretnej kolejce powiązanej z tym kluczem otrzymają wiadomość.
Tylko konsumenci z dokładnie takim samym kluczem routingu otrzymają wiadomość.

Z2 PRODUCER
Enter routing key:
direct.key1
Enter message:
Message for key1
Sent: Message for key1
Enter routing key:
direct.key2
Enter message:
Message for key2
Sent: Message for key2
Enter routing key:
exit

** Topic **
Producent wysyła wiadomość z kluczem routingu opartym na temacie (pattern),
a konsumenci oczekujący na wiadomości na kolejce powiązanej z odpowiednim kluczem routingu na podstawie tematu otrzymają wiadomość.
Wiadomości są filtrowane na podstawie zgodności wzorców kluczy routingu.

Z2 PRODUCER
Enter routing key:
topic.key1
Enter message:
Message for key1
Sent: Message for key1
Enter routing key:
topic.key2
Enter message:
Message for key2
Sent: Message for key2
Enter routing key:
exit

Z2 CONSUMER
Enter routing key:
*.key1
created queue: amq.gen-Q1w2E3r4T5y6
Waiting for messages...
Received: Message for key1
Enter routing key:
exit

Z2 CONSUMER
Enter routing key:
topic.*
created queue: amq.gen-Q1w2E3r4T5y7
Waiting for messages...
Received: Message for key1
Received: Message for key2
Enter routing key:
exit
